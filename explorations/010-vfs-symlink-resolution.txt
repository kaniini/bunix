Move symlink resolution policy into VFS.

Today some filesystem translators resolve symlink targets internally.  That is
the wrong long-term layering: VFS has the namespace context, mount routing, and
cross-translator visibility needed to resolve symlink chains consistently.

- [ ] Inventory every translator and Linux syscall path that currently depends on server-local symlink resolution.
- [ ] Teach VFS path walking to resolve symlink chains using `readlink` metadata from translators.
- [ ] Support relative symlink targets relative to the containing directory of the link.
- [ ] Support absolute symlink targets from the VFS namespace root, including targets crossing mount boundaries.
- [ ] Preserve `O_NOFOLLOW`, `AT_SYMLINK_NOFOLLOW`, and `readlink` behavior so callers can observe links without following them.
- [ ] Add a symlink loop limit and return Linux-compatible `ELOOP` through the Linux server.
- [ ] Refactor rootfs, tmpfs, unionfs, procfs, devfs, and future sysfs to expose links as metadata instead of resolving namespace policy internally.
- [ ] Add focused tests for absolute links, relative links, `..` components, mount-crossing links, nofollow stat, readlink, and loop detection.
