Fix host-side path safety in rootfs builders and test harnesses.

Problem:

- Host tooling accepts build/test paths from arguments and environment
  variables, then removes, overwrites, or imports those paths without a shared
  safety layer.
- Historical `mkrootfs --tree` walked a host directory with `lstat()`, but
  later opened regular-file sources by path.  A changing or malicious tree
  could race the lstat/open boundary and cause the generated rootfs image to
  include data from a different host file than the one audited.  Current
  state: `tools/mkrootfs.c` was retired by `4bed5e3 Retire legacy rootfs
  format`, so the remaining action is to keep that legacy importer out of the
  build graph and harden the active SquashFS staging scripts.
- Test harnesses and image builders perform `rm -rf` on caller-controlled
  runtime directories without the guard already present in
  `tools/test-prune-artifacts.sh`.

Threat model:

- Tertiary attacker: malformed or malicious rootfs fixture trees, generated
  Alpine rootfs contents, or environment-controlled build/test paths used in
  local Bunix build automation.
- Impact candidates: host file disclosure into generated Bunix rootfs images,
  accidental or malicious deletion of host directories during tests/builds,
  corrupted release/test artifacts, and test-environment compromise.

Evidence:

- Historical evidence: `tools/mkrootfs.c` used `lstat()` during
  `import_tree_path()`, recorded regular-file source paths, and later copied
  file contents with `fopen()` in `copy_file()`.  Current evidence:
  `git log --diff-filter=D --summary -- tools/mkrootfs.c` shows the file was
  deleted in `4bed5e3`, and `rg --files | rg mkrootfs` finds no source file.
- `tools/build-alpine-rootfs.sh` removes `$stage` with `rm -rf "$stage"` where
  `stage` can be supplied through `ALPINE_ROOTFS_STAGE`.
- `tools/test-boot.sh`, `tools/test-command.sh`, and `tools/test-shell.sh`
  remove `$tmp` during cleanup, where `tmp` can be supplied through
  `BUNIX_TEST_RUNTIME_DIR`.
- `tools/build-ext2-test-image.sh` removes `$tmp` and overwrites `$out`
  without using a common safe path helper.

Root cause:

- Host tools do not have a common path-safety policy for scratch directories,
  generated artifacts, symlink handling, tree import, and cleanup.
- The retired `mkrootfs` importer imported mutable host paths rather than
  stable file descriptors or `openat()` traversal with no-follow checks.
  Current SquashFS builders stage files with shell tools and `mksquashfs`.

Fix plan:

- Add a small shared shell helper for refusing unsafe scratch/output paths:
  empty, `/`, `.`, repository root, home directory, and paths outside expected
  build/tmp prefixes unless explicitly allowed.
- Update test harness cleanup and rootfs/image builders to use the helper
  before every `rm -rf`, generated artifact overwrite, or recursive copy.
- Add a regression that proves the retired `mkrootfs` source/build path is not
  present, and focus active hardening on SquashFS staging scripts.
- Add tests for unsafe runtime-directory refusal and active rootfs tooling
  path safety.
- Document which environment variables are allowed to point outside `build/`
  or `/tmp`, and require an explicit override for dangerous paths.

Notes:

- Added `tools/path-safety.sh` as a shared POSIX shell helper for generated
  scratch/output paths.  It rejects empty paths, `/`, `.`, the repository root,
  the user home directory, and generated paths outside the repository `build/`
  tree or a tmp directory.  `BUNIX_PATH_SAFETY_EXTRA_PREFIXES` is available as
  an explicit escape hatch for controlled host tooling.
- Converted cleanup/output deletion in the boot, command, shell, prune,
  Alpine rootfs, RISC-V rootfs, synthetic SquashFS, ext2 image, and Linux
  syscall audit scripts to use `safe_rm_rf` or `safe_rm_f`.
- Added `tools/test-path-safety.sh` and `make test-path-safety` for direct
  refusal coverage.  The test verifies safe generated paths under `/tmp` and
  `build/`, refusal of empty, root, current-directory, repository-root,
  home-directory, and non-generated relative paths, absence of the retired
  `mkrootfs` importer from live build/tooling references, and absence of new
  unguarded `rm -rf "$..."` sites in host shell tools.
- Unsafe cleanup reproducers now fail before deletion: `BUNIX_TEST_RUN_ROOT=/
  sh tools/test-prune-artifacts.sh`, `SQUASHFS_ROOTFS_STAGE=/ sh
  tools/build-synthetic-squashfs-rootfs.sh /tmp/bunix-path-safety.sqfs`, and
  `TMPDIR=/ sh tools/build-ext2-test-image.sh /tmp/bunix-path-safety.ext2`.
- Focused verification passed: `make test-path-safety`.
