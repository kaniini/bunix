Implement deterministic external-network smoke tests for VirtIO-net.

Problem:

- Exploration 018 brings up a userspace VirtIO-net driver and NET0
  packet-interface boundary, but it intentionally stops short of proving real
  external traffic.
- External smoke tests need more than a NIC driver: NET0 must produce and
  consume Ethernet frames, resolve neighbors, and the test harness must own a
  deterministic host-side packet peer.

Dependencies:

- Requires the VirtIO-net driver and packet-interface route state from
  `018-virtio-net.txt`.
- Requires IRQ-driven or otherwise reliable completion delivery from
  `028-virtio-irq-driven-completions.txt`, or a conscious decision that bounded
  polling is still acceptable for the first smoke target.

Constraints:

- Keep the NIC driver in userspace and keep Linux socket semantics out of the
  driver.
- NET0 owns Ethernet, ARP, IPv6 neighbor discovery, route lookup, packet
  filtering, and protocol-stack state.
- The host-side topology must be deterministic and harness-owned.  Do not rely
  on ambient host network configuration.

Tasks:

- [x] Choose the first deterministic host backend: isolated QEMU socket,
      tap+namespace, or constrained QEMU user networking with explicit packet
      probes.
- [ ] Add Ethernet encapsulation and decapsulation in NET0 for packet-interface
      routes.
- [x] Implement ARP for IPv4 on packet interfaces, including request, reply,
      cache state, timeout, and `/proc/net/arp` observability.
- [x] Implement enough IPv6 neighbor discovery for link-local and localhost-safe
      smoke tests, or explicitly defer IPv6 external traffic with a documented
      test gap.
- [ ] Add packet-interface egress queueing from UDP/TCP route lookup into
      `BUNIX_NET_PACKET_TX_ENQUEUE`.
- [ ] Add packet-interface ingress dispatch from `BUNIX_NET_PACKET_RX_SUBMIT`
      into Ethernet, ARP/ND, IPv4, IPv6, UDP, and TCP paths.
- [ ] Add host-side traffic generator support for deterministic RX and TX
      frames.
- [ ] Add smoke tests for ARP, IPv4 UDP egress/ingress, IPv4 TCP connect or
      accept, route fallback behavior, `/proc/net/dev` counters, and clean
      guest poweroff.
- [ ] Add equivalent IPv6 smoke tests once neighbor discovery is present.
- [ ] Document remaining production-networking work after the first smoke tests:
      DHCP/SLAAC, DNS, firewall/filtering, packet capture, offload negotiation,
      multiqueue policy, and performance counters.

Backend decision:

- Use QEMU user networking as the first external smoke backend, but keep it
  explicit instead of ambient: `make test-boot-virtio-net-external-ping` uses
  `QEMU_VIRTIO_NET_EXTERNAL_ARGS` with unrestricted user networking, boots the
  Alpine rootfs, obtains the current deterministic QEMU-user IPv4 lease through
  the userspace `udhcpc` path, pings `4.2.2.1` and `8.8.8.8`, checks NET0
  config, checks `/proc/net/dev` has the `eth0` interface, and powers off.
- Keep the default VirtIO-net boot and service tests on restricted QEMU user
  networking.  Those tests validate device bringup, NET0 config, OpenRC
  networking service behavior, and drained packet queues without depending on
  external host reachability.
- Treat QEMU user networking as the phase-1 backend only.  It is good enough to
  prove the Bunix userspace/NIC/NET0/Linux-personality path can exchange real
  packets, but it is not enough to prove full Ethernet behavior because QEMU's
  slirp backend can accept simplified broadcast IPv4 frames.  ARP/ND, UDP/TCP
  ingress/egress, and route fallback still need a harness-owned peer such as a
  tap+namespace or socket-backed packet generator.

Current implementation gaps:

- NET0 has packet-interface TX/RX queues and an external IPv4 ICMP send path.
  External ICMP and UDP currently build Ethernet+IPv4 frames in NET0 and
  enqueue them to VirtIO-net.  Unresolved next hops are held behind ARP
  resolution instead of being sent as broadcast IPv4 frames.
- Neighbor RPCs and `/proc/net/arp` observability now exist for IPv4.  NET0 can
  learn from IPv4 ARP frames and answer ARP requests for local IPv4 addresses,
  generates ARP requests for unresolved IPv4 next hops, and queues external
  IPv4 ICMP egress while ARP resolution is pending.  IPv4 neighbor entries are
  aged from the NET0 cache.  IPv6 ND is not implemented yet.
- External UDP route lookup now feeds packet-interface egress for IPv4,
  including ARP-backed queueing, and packet-interface IPv4 UDP ingress is
  dispatched into bound UDP sockets.  External TCP route lookup still does not
  feed packet-interface egress or ingress; localhost UDP/TCP remains covered by
  the core networking tests.
- The first deterministic host-side packet generator is still unimplemented.
  The next slice should choose between tap+namespace and socket-backed frames
  based on what gives the fastest repeatable RX/TX probes under `make test`.

Neighbor/ARP progress:

- NET0 now has working neighbor table RPCs for IPv4:
  `BUNIX_NET_NEIGHBOR_COUNT`, `BUNIX_NET_NEIGHBOR_AT`,
  `BUNIX_NET_NEIGHBOR_ADD`, and `BUNIX_NET_NEIGHBOR_DELETE`.
- `/proc/net/arp` is exposed by procfs from NET0 neighbor records, using a
  Linux-style ARP table format.  `nettest` verifies the file exists, and the
  bootstrap NET self-test verifies neighbor add/enumerate/delete behavior.
- Incoming IPv4 Ethernet frames continue to opportunistically learn source
  IPv4/MAC pairs into the neighbor table.  External IPv4 ICMP uses a learned
  neighbor for the next hop when one exists, and otherwise queues the frame
  behind ARP resolution.
- NET0 now consumes IPv4 ARP requests/replies, learns sender IPv4/MAC pairs,
  and answers ARP requests for local IPv4 addresses.  The bootstrap NET
  self-test injects a synthetic ARP request and verifies the generated ARP
  reply through the packet TX queue.
- NET0 now generates ARP requests before sending external IPv4 ICMP through an
  unresolved next hop.  The bootstrap NET self-test drives `BUNIX_NET_ICMP_SENDTO`
  to an unresolved on-link IPv4 address and verifies the ARP request frame
  through the packet TX queue.
- NET0 now queues external IPv4 ICMP frames while neighbor resolution is
  pending and flushes them with the resolved destination MAC when an ARP reply
  is learned.  The bootstrap NET self-test injects the ARP reply and verifies
  the queued ICMP frame is emitted with the resolved MAC instead of broadcast.
- NET0 now timestamps learned/manual IPv4 neighbor records and prunes stale
  entries during neighbor lookup and observation, so expired neighbors force
  ARP resolution again.
- Full ARP for the ICMP path is now present enough for the current smoke scope;
  the broader exploration still needs UDP/TCP packet-interface egress/ingress
  and a harness-owned packet peer.

UDP packet-interface progress:

- IPv4 UDP send/connect now accepts routed non-local destinations.  NET0 builds
  Ethernet+IPv4+UDP frames, binds an ephemeral source port for unbound sockets,
  resolves the next hop with the IPv4 neighbor cache, and queues the UDP frame
  while ARP resolution is pending.
- The bootstrap NET self-test opens an IPv4 UDP socket, sends to a synthetic
  non-local address, verifies the generated ARP request, injects an ARP reply,
  and verifies the queued UDP frame is emitted with the resolved MAC, IPv4
  protocol 17, expected UDP destination port, length, and payload.
- The same bootstrap test binds a second IPv4 UDP socket, injects a synthetic
  Ethernet+IPv4+UDP frame addressed to that socket, verifies poll readiness,
  and receives the expected source port and payload through the NET0 UDP API.

IPv6 external-traffic deferral:

- External IPv6 packet-interface traffic is explicitly deferred from the first
  VirtIO-net external smoke target.  Loopback IPv6 remains covered by the core
  networking tests, but NET0 does not yet implement Ethernet IPv6 egress,
  router solicitation/advertisement, neighbor solicitation/advertisement,
  duplicate-address detection, or SLAAC.
- The deferred IPv6 work should add ND cache state and `/proc` observability in
  the same style as IPv4 ARP, then add equivalent packet-interface IPv6 UDP,
  ICMPv6, and TCP smoke tests with a deterministic host-side peer.
