Clean up Linux exec and shebang handling.

The x86 syscall path currently owns too much Linux personality policy:
argument collection, environment collection, shebang parsing, ELF loading,
interpreter loading, auxv construction, proc registration, and command-line
publication all live in `arch/x86_64/user.c`.  That is useful for early
bringup, but it is the wrong long-term boundary for a portable microkernel.

This exploration should preserve the working behavior from
`007-openrc-linux-compat.txt`, then move policy out of arch code in small
testable pieces.

Additional finding:

- After Alpine OpenRC reaches login, `/sbin/rc-status` can fail from the shell
  with `Out of memory` even though the VM server still reports many free
  frames.  This points at the current kernel-side Linux `execve()` loader's
  full-file buffering and contiguous large slab allocations for dynamic
  binaries/interpreters, not at physical memory exhaustion.  Preserve OpenRC
  startup behavior, but make the exec refactor address this loader allocation
  shape rather than baking it in.

- [ ] Inventory the current `arch/x86_64/user.c` exec path and separate
      architecture mechanics from Linux personality policy.
- [ ] Define an exec request/reply protocol between the syscall entry path,
      Linux server, proc server, and VM/task primitives.
- [ ] Move shebang parsing and argv rewriting out of arch code while keeping
      Linux-compatible interpreter argument semantics.
- [ ] Move ELF and PT_INTERP policy toward shared proc/Linux-server helpers,
      leaving arch code responsible only for final register/stack activation
      details that are truly architecture-specific.
- [ ] Replace full executable/interpreter image buffering with segment-oriented
      loading so later dynamic execs do not depend on large contiguous kernel
      slab allocations.
- [ ] Preserve `AT_EXECFN`, argv, envp, auxv, credentials, process identity,
      and Bunix task/Linux PID mappings across the refactor.
- [ ] Add regression tests covering ELF exec, dynamic PT_INTERP exec,
      shebang exec with interpreter arguments, recursive shebang limits, and
      invalid script diagnostics.
- [ ] Keep the resulting design portable enough for the future ARM port.
