Add apk to the Alpine image and verify package installation.

Problem:

- The Alpine rootfs should include `apk` so the system can install packages
  from inside Bunix.
- It is not enough for the binary to exist; package installation needs working
  DNS/networking, TLS or repository fetch behavior as configured, filesystem
  writes, directory metadata, temporary files, and process execution.

Intended outcome:

- The default Alpine image used by `make run` contains `apk`.
- After login, an `apk add` smoke test can initialize or use the package
  database and install a small package successfully.
- Failures in the install path are traced to concrete missing syscalls,
  filesystem semantics, networking behavior, or image construction issues.

Tasks:

- [x] Update the Alpine image construction so `apk` is present in the default
      environment.
- [x] Verify `apk --version` and basic package database access inside Bunix.
- [x] Run a minimal `apk add` smoke test against the configured Alpine
      repositories.
- [x] Fix any missing syscall, VFS, networking, temporary-file, or metadata
      behavior needed by the smoke test.
- [x] Add automated coverage for the apk presence check and at least one
      deterministic package installation path.

Notes:

- The default Alpine rootfs now includes `apk-tools` and `alpine-keys`, writes
  guest repository configuration, and validates those files in
  `test-alpine-rootfs`.
- Added `tools/guest-apk-install-smoke.sh` and
  `make test-boot-virtio-net-apk-add`.  The guest test verifies OpenRC
  networking state, resolver configuration, apk presence, package database
  access, DNS, repository fetch, and `apk add busybox-extras`.
- The apk install path needed several compatibility fixes:
  - Linux `fchdir()` now opens an independent VFS directory handle for the new
    cwd instead of aliasing a process fd handle.
  - Linux `O_TMPFILE` is emulated with a hidden upper-layer tmpfile, and
    `linkat(AT_SYMLINK_FOLLOW, /proc/self/fd/N, ...)` can commit that file.
  - Kernel exec resolves `/proc/self/fd/N` through the Linux server so apk
    maintainer scripts can run fd-backed temporary scripts.
  - Unionfs copies up lower regular files before metadata mutation, rename, and
    hardlink operations needed by apk database commits.
  - Tmpfs unlink/rmdir now reports `NOENT` when the parent path is absent
    instead of leaking a misleading permission error.

Validation:

- `make test-alpine-rootfs`
- `make test-boot-virtio-net-apk-add`
