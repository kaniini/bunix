Implement IPv6 neighbor discovery and deterministic external IPv6 smokes.

Problem:

- Exploration 025 intentionally deferred external IPv6 packet-interface traffic
  while bringing up the first deterministic VirtIO-net IPv4 smoke path.
- NET0 needs real IPv6 Ethernet behavior before IPv6 external smokes can be
  treated as equivalent to the IPv4 ARP/ICMP/UDP/TCP coverage.

Dependencies:

- Builds on `025-virtio-net-external-smoke.txt`.
- Requires the QEMU sidecar harness and `tools/virtio-net-peer.py` or a
  successor host-side peer that can generate and consume IPv6 Ethernet frames.

Constraints:

- Keep IPv6 neighbor discovery in NET0, not in the VirtIO-net driver.
- Keep Linux socket compatibility layered on NET0 sockets and RPCs.
- Use deterministic host-owned packet peers; do not depend on ambient host
  network configuration.

Tasks:

- [x] Add IPv6 Ethernet encapsulation and decapsulation in NET0 for
      packet-interface routes.
- [x] Implement ICMPv6 neighbor solicitation and neighbor advertisement parsing
      and generation.
- [x] Add IPv6 neighbor cache state, timeout policy, and `/proc` observability
      equivalent to `/proc/net/arp`.
- [x] Add router solicitation/advertisement parsing sufficient for deterministic
      smoke tests, or document an explicit static-route-only first phase.
- [x] Add duplicate-address-detection behavior or explicitly defer it with
      tests proving the deferral is isolated from configured static addresses.
- [x] Add packet-interface IPv6 ICMP echo egress/ingress smokes with the
      host-side peer.
- [x] Add packet-interface IPv6 UDP egress/ingress smokes with the host-side
      peer.
- [x] Add packet-interface IPv6 TCP SYN connect/accept smokes with the
      host-side peer.
- [x] Add `/proc/net/dev` and neighbor-table counter checks for IPv6 RX/TX.
- [x] Add clean guest poweroff coverage for the deterministic IPv6 smoke target.

Non-goals for the first slice:

- Full production SLAAC, DNS resolver updates, firewalling, and multiqueue
  policy remain production-networking follow-up work unless pulled into this
  exploration explicitly.

Progress:

- NET0 now has IPv6 route-prefix matching, packet-interface ICMPv6 Ethernet
  egress/ingress, NDP neighbor solicitation/advertisement parsing and
  generation, IPv6 pending-frame flush behind neighbor resolution, and
  opportunistic IPv6 neighbor learning.
- `/proc/net/ndisc` exposes IPv6 neighbor records while `/proc/net/arp` remains
  IPv4-compatible.
- NETCFG installs a deterministic static `2001:db8:18::15/64` packet-interface
  address and on-link route for the current socket-peer smoke path.  Router
  solicitation/advertisement is explicitly deferred for this static-route-only
  phase.
- `tools/virtio-net-peer.py` now answers IPv6 NDP and ICMPv6 echo on the QEMU
  socket multicast backend.
- `make test-boot-virtio-net-socket-peer-ipv6` passed, covering BusyBox
  `ping -6 2001:db8:18::2`, `/proc/net/ndisc`, `/proc/net/dev` counters, and
  clean guest poweroff.
- NET0 now supports IPv6 UDP packet-interface egress and ingress using the
  same route, neighbor-cache, NDP, and pending-frame path as ICMPv6.  The host
  peer echoes UDPv6 datagrams, and `make test-boot-virtio-net-socket-peer-udp6`
  passed with `/bin/nettest udp6-external`.
- NET0 now supports the same minimal external TCP SYN scope for IPv6 that the
  IPv4 smoke path has: routed connect emits an Ethernet+IPv6+TCP SYN through
  NDP-backed packet-interface egress, and inbound IPv6 TCP SYN frames create an
  accepted socket and emit SYN-ACK.  `make test-boot-virtio-net-socket-peer-tcp6`
  passed with `/bin/nettest tcp6-external-connect` and
  `/bin/nettest tcp6-external-accept`.
- Duplicate-address detection is explicitly deferred for this static-route
  phase.  The deferral is isolated from configured static addresses by the
  socket-peer ICMPv6, UDPv6, and TCPv6 smokes: NETCFG installs
  `2001:db8:18::15/64` directly, NET0 uses it for source address selection, and
  the deterministic peer resolves and exchanges traffic with that address
  without requiring tentative address state or SLAAC.
