Problem:

- The current riscv64 boot package is a smoke carrier containing
  `bootstrap`, `abi-smoke.user`, `linux`, and `/bin/musl-hello`.
- Alpine-on-riscv64 needs a reusable carrier for the normal server graph plus
  rootfs content, not a fixed static-hello package.
- The Makefile has hand-written riscv64 module rules that duplicate host-side
  server packaging concerns and bake in the riscv64 stub Linux server.
- Exploration 035 prepared a host-side riscv64 Alpine rootfs artifact and
  dynamic-linker overlay artifacts.  This exploration should consume that work
  rather than rediscovering rootfs construction.
