Implement VFS-backed mmap metadata and file-hole SIGBUS delivery.

Problem:

- The Linux mmap path currently allocates anonymous user pages and eagerly
  copies file data into them.
- Once bytes are copied, the VM layer no longer knows which pages came from a
  file, what file extent they represent, or whether later access is past the
  mapped file object.
- Linux-compatible file-backed mappings should deliver SIGBUS for invalid
  file-backed accesses such as mapping holes or pages beyond the valid file
  object, instead of silently exposing zero-filled anonymous memory or reporting
  an ordinary SIGSEGV.

Constraints:

- Keep filesystem policy in VFS/filesystem servers.  The kernel VM layer should
  hold enough object metadata to ask the owning service for page contents or
  fault classification, but should not learn filesystem-specific layout rules.
- Preserve anonymous/private mmap behavior that is already sufficient for
  dynamic linking and current BusyBox workloads.
- Keep the design portable: x86 page-fault classification should hand off a
  generic file-backed mapping fault class that ARM can produce later.

Tasks:

- [x] Inventory current Linux mmap, VFS read, VM region metadata, task clone,
      munmap, mprotect, and fault classification paths.
- [x] Extend task VM region metadata to represent file-backed mappings with
      file identity, file offset, mapping length, protections, and private/COW
      policy.
- [x] Replace eager file mmap copying with either fault-time population or
      explicit VFS-backed mapping metadata that can distinguish EOF zero-fill
      from file-hole SIGBUS cases.
- [x] Add a Linux-server/VFS RPC contract for classifying file-backed mmap page
      faults and reading the relevant page contents.
- [x] Route file-backed mapping holes through the task-fault event path as
      SIGBUS with a useful `si_addr` and Linux-compatible `si_code`.
- [x] Add regressions for file mappings that stay inside file size, partial
      final-page zero-fill where Linux permits it, and invalid file-backed
      access that must deliver SIGBUS.
- [x] Re-run focused dynamic-linker, mmap, signal, and boot tests.

Completion criteria:

- Static and dynamic userspace continue to boot and run.
- A focused test can catch SIGBUS from a file-backed mmap hole and verify
  `siginfo_t.si_addr`.
- Existing SIGSEGV/SIGBUS default and caught-signal tests still pass.

Implementation Notes:

- Inventory result: x86 and riscv64 Linux `mmap` paths were allocating full
  anonymous ranges and eagerly copying file bytes through the existing Linux
  server `BUNIX_LINUX_MMAP` read RPC.  `task_vm_region` already carried object
  type, object id, offset, length, flags, and protections, and fork/munmap/
  mprotect already preserved that metadata.  The missing piece was that file
  mappings were registered as anonymous after the eager copy.
- File-backed Linux `mmap` now registers VMAs as `TASK_VM_OBJECT_FILE` with
  the fd as the current object id and the mmap offset as the object offset.
  Anonymous mappings remain `TASK_VM_OBJECT_ANON`.
- File mmap still uses eager population for now, but it no longer leaves full
  pages beyond EOF mapped as anonymous zero pages.  The mmap helper tracks the
  number of bytes actually read, preserves Linux's permitted zero-filled
  partial final page, and unmaps subsequent full pages so later access faults.
- This is not yet a lazy pager: classification is performed during mmap
  population instead of by calling VFS on the actual hardware page fault.
- x86 user page-fault classification now asks generic task VMA metadata whether
  a non-present fault is inside a file-backed mapping.  Those faults are sent
  to the Linux server as `BUNIX_TASK_FAULT_CLASS_BUS`, while ordinary missing
  mappings remain SIGSEGV/MAPERR.
- x86 signal-frame construction now distinguishes alignment-check SIGBUS
  (`BUS_ADRALN`) from file-backed mapping SIGBUS (`BUS_ADRERR`) and preserves
  the faulting `si_addr`.
- The existing `signaltest` regression now also maps `/hello.txt` for two
  pages, verifies the first byte is readable, verifies the partial final page
  zero-fill at byte 4095, and requires `SIGBUS`/`BUS_ADRERR` with `si_addr`
  equal to the first address in the second page.
- Added `BUNIX_VFS_MMAP_PAGE_BUFFER`, returning explicit page classes:
  `DATA`, `ZERO`, and `BUS`.  The central VFS server and unionfs route the op
  through existing open handles, and tmpfs, squashfs, and ext2 implement it.
  Tmpfs returns dense data or BUS past EOF, squashfs can report explicit stored
  zero blocks as ZERO, and ext2 currently materializes readable extents through
  its existing zero-aware read path.
- The Linux server now uses the mmap-page VFS op for file mmap population and
  returns the page class to the arch mmap helper.  Arch mmap code keeps valid
  ZERO pages mapped as zero-filled pages and leaves BUS pages unmapped so the
  task-fault path delivers SIGBUS on access.
- The VFS-backed page classification keeps eager page population for now, but
  the classification policy is no longer hardcoded in the arch mmap path.
- The remaining future improvement is a true lazy pager path that asks VFS at
  page-fault time instead of during mmap population; that is not required for
  this exploration's SIGBUS correctness criteria.
