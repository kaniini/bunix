Problem:

- `arch/riscv64/early.c` still performs many test assertions directly in the
  normal boot path: boot-layout diagnostics, platform-discovery diagnostics,
  scheduler self-test, timer wait, context-switch self-test, syscall self-test,
  U-mode self-test, boot-package parsing, module validation, ELF-loader
  self-test, and user-copy self-test.
- These checks are valuable for development, but an Alpine boot path needs a
  cleaner split between required initialization and opt-in diagnostics.

Progress notes:

- Created from the audit in
  `035-riscv64-cleanup-alpine-environment.txt`.
- Marker classification:
  - Required boot evidence remains unconditional: entry banner, PMM range
    marker, boot-package magic, boot-package command-line marker, module
    registration, module ELF validation, server launches, service
    registrations, proc/VFS/rootfs readiness, Linux smoke completion, and
    machine/SBI poweroff.
  - Hardware/firmware diagnostics now require `riscv64-diag`: boot memory
    layout, initrd/FDT layout, CPU count, SMP parked policy, timebase,
    stdout/UART details, UART count, and interrupt-controller/routing data.
  - Test-only self-tests now require `riscv64-selftest`: scheduler worker
    smoke, timer tick wait, context switch, VM hooks, synthetic syscall frame,
    U-mode trap return, generic ELF loader, and user-copy checks.
- The default early and UART packages include `riscv64-diag riscv64-selftest`;
  the Alpine package intentionally omits them.
- The old kernel-side riscv64 bootpkg poweroff latch is gone.  The smoke
  squashfs now contains `/bin/rv64-syscall-smoke` and `/bin/musl-hello`, and
  the riscv64 bootstrap registers, spawns, and waits for both through proc
  before logging `bootstrap-riscv64: done` and powering off.
- The UART package remains lean.  It carries `names`, `bootstrap`, and
  `abi-smoke.user`; it does not carry or launch the full shared server graph.
- Native sleep remains out of the early boot path.  Follow-up exploration
  `067-riscv64-native-sleep-timer-semantics.txt` tracks a proper native
  sleep/timer audit with a dedicated smoke target.
- During verification, running riscv64 boot targets in parallel exposed that
  they share `build/riscv64-serial.log`.  Follow-up exploration
  `066-riscv64-test-log-isolation.txt` tracks per-target log isolation.
- Additional package-shape verification:
  - `make test-riscv64-bootpkg`
- Full completion gate:
  - `make test` launched 20 shards with 0 skipped and all shards passing.
