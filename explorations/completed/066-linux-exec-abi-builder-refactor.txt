Refactor Linux exec argument, environment, auxv, and stack construction.

Parent exploration: `043-user-address-space-identity-map-escape.txt`

Problem:

- Linux exec currently spreads path/vector serialization, shebang rewriting,
  ELF loading, process bookkeeping, and user stack construction across a large
  Linux server file.
- Debugging login and shell startup regressions requires touching code that is
  easy to perturb and hard to validate in isolation.
- Future dynamic-linker, shebang, login, and OpenRC work will keep stressing
  this path, so it needs a tighter ABI builder with focused tests.

Why this is a refactor:

- The existing behavior should not change as part of this exploration.
- The goal is to extract the exec ABI rules into a smaller, testable module
  while preserving current Linux personality semantics.

Plan:

- Split exec vector serialization/deserialization into a dedicated Linux exec
  helper module.
- Split stack image construction into a helper that owns argc, argv, envp,
  auxv, random bytes, alignment, and string placement.
- Add host-side unit tests for common exec layouts, long vectors, empty env,
  shebang rewrites, and stack alignment.
- Add guest regression coverage for BusyBox applets, dynamic musl binaries,
  shebang scripts, login shell startup, and OpenRC helper scripts.
- Keep policy decisions in the Linux server; only move mechanical ABI building
  into helpers.

Progress:

- Current exec path ownership:
  - `arch/x86_64/user.c` handles the Linux `execve` syscall entry, copies the
    user path, argv, and envp out of the calling task, reads shebang prefixes,
    and forwards shebang vector preparation to the Linux server.
  - `user/linux/main.c` owns Linux personality policy for exec preparation,
    including vector deserialization/serialization, shebang parsing, shebang
    argv rewrite, credential collection, and RPCs to proc.
  - `user/proc/main.c` now owns ELF loading, interpreter loading, segment
    mapping, initial stack construction, auxv construction, and task image
    replacement.
- Extracted the Linux-server exec vector and shebang ABI helpers into
  `user/linux/exec_abi.c`, included by `user/linux/main.c`.
- Added `tools/test-linux-exec-abi.c` and `make test-linux-exec-abi` to cover:
  - argv/envp serialization and deserialization round-trips.
  - shebang interpreter and optional argument parsing.
  - shebang argv rewriting with preserved tail arguments.
  - rejection of invalid relative shebang interpreters.
- Extracted proc-server initial stack and auxv construction into
  `user/proc/exec_stack.c`, included by `user/proc/main.c`.
- Added `tools/test-proc-exec-stack.c` and `make test-proc-exec-stack` to
  decode the generated stack image and cover:
  - argc, argv, envp, and terminators.
  - ELF auxv entries for entry point, phdrs, interpreter base, and execfn.
  - credential auxv entries.
  - Bunix capability auxv entries for console, time, proc, and names.
  - default argv construction when no explicit vectors are supplied.
