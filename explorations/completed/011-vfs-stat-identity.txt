Fix VFS-backed Linux stat identity correctly.

OpenRC exposed that musl's dynamic linker de-duplicates DSOs using
`fstat()` `st_dev` and `st_ino`.  The current Linux personality reports
constant identity for many files, which can make distinct shared objects
look identical.  A temporary handle-derived inode is only a diagnostic
shortcut; this exploration should replace it with real VFS identity.

Related findings:

- See `007-openrc-linux-compat.txt` for the OpenRC loader trace that found
  musl opening both `/usr/lib/librc.so.1` and `/usr/lib/libeinfo.so.1` before
  still reporting unresolved `libeinfo` symbols.
- That trace, together with local `~/musl/ldso` source, is the motivating
  evidence for treating `st_dev`/`st_ino` identity as a correctness blocker
  for dynamic OpenRC userspace.

- [x] Define VFS metadata identity fields for filesystem device ID, inode ID,
      link count, block size, block count, and rdev where applicable.
- [x] Extend VFS stat metadata replies without reviving the old 16-byte VFS
      protocol or breaking existing translators.
- [x] Assign stable per-filesystem device IDs to rootfs, tmpfs, procfs, devfs,
      sysfs, utmpfs, unionfs, and mounted translator views.
- [x] Give each filesystem stable inode IDs with correct hard-link behavior:
      same inode for hard links, distinct inode for distinct files, and stable
      identity across open/stat/fstat for the same object.
- [x] Make unionfs preserve lower-layer identity for lower files, upper-layer
      identity for materialized files, and coherent identity after copy-up,
      rename, whiteout, and hard-link operations.
- [x] Update Linux `stat`, `fstat`, `newfstatat`, and `statx` to populate
      `st_dev`, `st_ino`, `st_nlink`, `st_rdev`, `st_blksize`, and block
      counts from VFS metadata.
- [x] Add focused tests that compare `stat()` and `fstat()` identity for
      `/usr/lib/librc.so.1`, `/usr/lib/libeinfo.so.1`, hard links, symlinks,
      tmpfs files, and unionfs lower/upper files.
- [x] Add a regression test proving musl does not collapse distinct OpenRC
      DSOs by inode identity and `/sbin/openrc --help` resolves `libeinfo`
      symbols.
- [x] Remove any handle-derived inode shortcuts and temporary loader-path
      diagnostics once real identity is in place.
- [x] Run focused stat/loader tests, then the OpenRC Linux compatibility shard.
