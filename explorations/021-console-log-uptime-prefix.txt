Prefix console log messages with uptime.

Priority:

- Run this before the security vulnerability audit so the audit has stable,
  timestamped boot, server, and dmesg evidence to work from.

Problem:

- Bunix console log messages are useful, but once many servers are active it
  is hard to tell ordering, latency, and time gaps from text alone.
- Linux-style log timestamps make boot, OpenRC, scheduler, IPC, server, and
  dmesg investigations much easier without requiring full tracing.
- The timestamping should apply to log messages, not raw interactive console
  writes, so shell output and terminal control sequences remain unchanged.

Design direction:

- Add a monotonic uptime prefix to every complete console log line, similar to
  Linux `dmesg` output.
- Keep raw console writes used for TTY/user output unprefixed.
- Use the kernel's monotonic time source where available, with a sensible early
  boot behavior before the timer is fully online.
- Make the format stable enough for tests while avoiding overfitting existing
  marker checks to a single timestamp value.

Implemented format:

- Log records are prefixed as `[seconds.useconds] `, for example
  `[0.120000] kernel: idle`.
- Precision is microseconds derived from the generic kernel
  `timer_monotonic_ns()` API.
- Prefixing is line-oriented on the kernel log stream.  The first byte after
  boot or after `\n` emits one prefix.  Fragmented server log writes continue
  the current line until a newline arrives, so chunking does not double-prefix
  one logical log line.
- Raw console/TTY writes stay unprefixed.

Constraints:

- Do not break existing serial marker tests.  Tests that look for substrings
  should keep working, but exact-line checks may need targeted updates.
- Prefix only log records emitted through console log paths, including server
  `bunix_console_log()` output and dmesg-visible kernel log records.
- Preserve the distinction between early console, console server output, ring
  buffer logging, and user-facing terminal writes.
- Make the implementation SMP-safe and compatible with existing console
  locking.
- The timestamp source should be architecture-neutral enough to survive future
  ARM work.

- [x] Define the exact uptime prefix format, precision, early-boot fallback,

- [x] Implement line-oriented prefixing in the kernel console log path so each

- [x] Ensure `bunix_console_log()` from userspace servers is prefixed exactly
