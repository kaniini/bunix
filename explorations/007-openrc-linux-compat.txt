Close Linux personality gaps discovered by OpenRC userspace.

Findings:

- Alpine reaches OpenRC startup and `/sbin/openrc` runs far enough for musl's
  dynamic linker to search for dependencies.
- The dynamic linker opens `/usr/lib/librc.so.1` and `/usr/lib/libeinfo.so.1`,
  but relocation still reports missing `libeinfo` symbols.
- Local musl `ldso` source shows loaded DSOs are de-duplicated with `fstat()`
  `st_dev`/`st_ino`; Bunix currently reports constant Linux stat identity for
  many files, so distinct shared objects can look identical.
- Fixing `/sbin/openrc` dependency loading should therefore wait for
  `011-vfs-stat-identity.txt` rather than using handle-derived inode shortcuts.
