Fix OpenRC DSO loading under generated Alpine.

Status:

- Planned as a prerequisite for completing `054-openrc-runtime-state.txt`.
- This is not a `/run/openrc` runtime-state slice.  Keep it focused on why
  musl's dynamic loader rejects OpenRC's shared libraries.

Problem:

- The generated Alpine image reaches login, but `/sbin/openrc --help` fails.
- Musl reports:
  - `Error loading shared library librc.so.1: Invalid argument (needed by /sbin/openrc)`
  - `Error loading shared library libeinfo.so.1: Invalid argument (needed by /sbin/openrc)`
- The resulting unresolved OpenRC symbols make the boot probe fail before it
  can be used as strong evidence for the runtime-state behavior in
  exploration 054.
